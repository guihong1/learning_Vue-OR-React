🔍 嵌套路由的特点与优势

✅ 结构清晰

  • 路由层级对应页面结构
  • URL 路径直观易懂
  • 便于维护和扩展

✅ 组件复用

  • 父组件提供公共布局
  • 子组件专注具体功能
  • 减少代码重复

✅ 导航便利

  • 支持多级导航菜单
  • 面包屑导航自动生成
  • 路由状态管理简单

✅ 权限控制

  • 可在不同层级设置守卫
  • 细粒度的权限管理
  • 灵活的访问控制

⚡ 实现要点与注意事项

重要提示：
  • router-view 嵌套：父组件必须包含 <router-view> 来渲染子路由
  • 路径匹配：子路由的路径是相对于父路由的
  • 默认子路由：使用空字符串 '' 作为默认子路由路径
  • 重定向处理：可以使用 redirect 属性设置默认跳转
// 嵌套路由完整示例
const routes = [
{
path: '/admin',
component: AdminLayout, // 父组件
children: [
// 默认子路由：/admin
{ path: '', component: AdminHome },
// 子路由：/admin/users
{ path: 'users', component: UserManagement },
// 嵌套子路由：/admin/users/123
{ path: 'users/:id', component: UserDetail }
]
}
]
